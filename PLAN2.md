## 1. What v2 changes and why

v1 treated the MT construction session as the whole app. v2 makes it one room in a larger house. The app becomes four user-chosen modes — **Learn, Speak, Read, Listen** — over a shared FSRS/mastery spine, with **Learn as the centerpiece** and a recommended daily path stitching the modes together. Hard gating is replaced by soft recommendation: everything is browsable, order is suggested, nothing is locked except where prerequisites make content genuinely unusable.

**Product goal restated honestly**: a two-week intensive gets the user to functional everyday-France competence (order, shop, navigate, transact, react, follow fast replies) in local register; the same system continues toward long-term fluency. The app never claims fluency-in-two-weeks.

### Non-negotiable invariants (apply to every mode, every phase)
1. Every gradeable interaction in any mode emits a `DrillEvent` and updates FSRS/mastery through `MasteryModel.recordDrill`. One spine, four doors.
2. Vocabulary is never drilled as bare word lists — always inside sentences.
3. Street register is first-class everywhere: every French string that has a distinct spoken form carries it, and Speak/Listen default to street register at realistic speed.
4. All content is batch-generated by the pipeline and shipped in the pack. No live API calls in the app in v2.
5. Design language per PLAN.md §8. No gamification. The DSMotion spring baseline on all state transitions.

## 2. Information architecture

```
Home
├── Recommended today (intensive-path card: 1 Learn lesson + 1 Speak scenario + 1 Listen episode)
├── LEARN  (centerpiece — largest section)
│   ├── Construction   (the v1 MT session, renamed; unchanged engine)
│   ├── Conjugation    (verb pattern families, drilled)
│   ├── Vocabulary     (frequency-ranked packs, drilled in sentences)
│   └── Grammar        (focused rule lessons + attached drills)
├── SPEAK  (everyday-France scenarios, branching dialogues)
├── READ   (generated passages emulating broad real-world source styles)
└── LISTEN (comprehension gym: fast street dialogues, staged reveal)
```

Navigation: Home is a scrolling library (see §8), each mode opens a browsable index, each content unit opens a player/reader. The v1 TodayView is replaced.

## 3. Content specification (pipeline v2)

All new content lands in `content_pack_v2/` (v1 pack remains, still consumed for Construction). Same pipeline pattern: generate → validate → TTS → assemble. All generation uses claude-sonnet-4-6; all audio Google WaveNet fr-FR (street regeneration stays BACKFILL for ElevenLabs). New: **English prompt audio** (Google TTS en-US standard voice) for every drill prompt in v1 and v2 packs — this resolves the hands-free BACKFILL.

### 3.1 Conjugation module
- **Verbs (20)**: être, avoir, aller, faire, vouloir, pouvoir, devoir, savoir, prendre, venir, dire, voir, mettre, parler, aimer, acheter, manger, partir, finir, donner.
- **Tenses v2**: présent, passé composé, imparfait, futur proche. Plus conditional politeness forms (je voudrais / je pourrais / il faudrait) as a mini-module.
- Structure: verbs grouped into pattern families (regular -er / -ir / -re; then irregulars individually). Each verb unit = conjugation table (with audio per form, formal + street where they differ, e.g. *tu as* → *t'as*) + 15–20 drill sentences per verb exercising the forms in context.
- Node representation: each verb unit is a ConceptNode of new type `conjugation`, tier assigned by frequency; drills are Sentences with `targetConceptId` = the verb node.

### 3.2 Vocabulary module
- **Corpus**: top 1,000 French lemmas by spoken frequency (seed from an open frequency list, e.g. Lexique/OpenSubtitles-derived; the generation prompt receives the ranked list — pipeline includes it as `data/freq_fr_1000.txt`, hand-checked for obvious junk).
- Structure: 40 packs × 25 words, frequency-ordered. Each pack is a ConceptNode of type `vocab_pack`. Each word appears in ≥2 generated drill sentences (formal + street + English), sentences constrained to previously-introduced grammar where feasible (best-effort, validator warns rather than rejects for vocab packs).
- Dedupe against words already taught by v1 concepts and conjugation module.

### 3.3 Grammar module
- **15 lessons**: gender & articles; plurals & adjective agreement; adjective placement; prepositions à/de/en/au/chez; question forms (deep dive); negation variants (ne...jamais/rien/plus/personne, street drops); object-pronoun order; y & en deep dive; comparatives & superlatives; imperatives; reflexive verbs; partitive quantities (du/de la/des); il faut & impersonal expressions; time & date expressions.
- Each lesson = ConceptNode type `grammar`: a ≤200-word explanation written in the MT conversational voice, 6–8 canonical examples (formal + street + English, with audio), 20 drill sentences.

### 3.4 Speak scenarios
- **12 scenarios**, everyday-life-in-France priority order: (1) café order, (2) restaurant full meal, (3) boulangerie/market, (4) clothing shop incl. sizes & paying, (5) métro/train & tickets, (6) asking & following directions, (7) hotel/Airbnb check-in & issues, (8) pharmacy & minor ailments, (9) phone calls & reservations, (10) small talk with strangers, (11) bar/social evening, (12) problems & complaints (returns, wrong order, lost item).
- Structure per scenario: a **branching scripted dialogue**, 8–14 exchanges deep, 2–3 branch points (e.g. the waiter asks a follow-up you must handle), 3 generated **variants** per scenario so replays differ. User lines carry: English intent prompt, target French (street primary, formal alternate), audio. NPC lines carry: French (street register), audio at `street_fast` + `street_slow`, English gloss.
- JSON schema: `scenario.json` = {id, title, icon, setting_blurb, variants: [{variant_id, nodes: [{node_id, speaker: npc|user, english, french_street, french_formal?, audio_refs, next | branches: [{label_english, next}]}]}]}.
- Every user line grades (self-grade v2; speech recognition later) → DrillEvent axis `production`.

### 3.5 Listen episodes
- **30 episodes**, 4 difficulty levels (A: slow-ish street, short → D: full-speed street with fillers/reductions, 60–90s).
- Each: a 2-person dialogue on everyday topics (weekend plans, complaining about the métro, gossip, ordering, phone call), street register with natural fillers. Assets: full audio (fast), full audio (slow), per-line transcript with English gloss, 3 comprehension questions (tap-answer) → DrillEvent axis `comprehension`.
- Two voices: use two distinct WaveNet fr-FR voices for speaker separation.

### 3.6 Read passages
- **60 passages**, generated but emulating broad real-source styles for wide exposure: news blurb, café/restaurant review, text-message exchange between friends (heavy street register), social-media post, travel blog excerpt, short fiction vignette, recipe, practical email (booking, complaint), event listing, opinion snippet.
- 4 difficulty tiers × mixed styles. Each passage: 60–200 words, per-word gloss map (JSON: {surface_form: english_gloss} for content words), 2–3 comprehension questions → DrillEvent axis `comprehension`. Text-only in v2 (no passage audio — keeps TTS budget sane; note as future).
- Style diversity is a validation criterion: no more than 8 passages per style.

### 3.7 Pipeline mechanics & budget
- New scripts in `content-pipeline/`: `gen_conjugation.py`, `gen_vocab.py`, `gen_grammar.py`, `gen_scenarios.py`, `gen_listen.py`, `gen_read.py`, `gen_english_prompts.py` (covers v1+v2), shared `tts.py` refactor, `assemble_pack_v2.py`, extended `validate.py` (schema validation per content type; DAG checks for new concept nodes; style-diversity check for passages).
- Estimated TTS volume: conjugation ~40k chars, vocab ~120k, grammar ~35k, scenarios ~45k, listen ~50k, English prompts ~80k ≈ **370k chars ≈ 37% of the monthly free tier**. If a run risks the cap, `tts.py` supports `--budget-check` dry run and the run splits across a month boundary. Claude generation cost estimate: $5–10 total.
- Everything resumable and idempotent like v1.

## 4. Data model deltas (SwiftData)

Extend, don't rewrite. Migration: additive fields with defaults + new models — lightweight migration should suffice; if not, importer rebuilds from packs (all state that matters to preserve: DrillEvents, MasteryScores, Sentence FSRS fields — write a migration test).

- `ConceptType`: add cases `conjugation`, `vocab_pack`, `grammar`.
- `Sentence`: add `englishAudioRef: String?`, `packVersion: Int` (1 or 2). (`targetConceptId` exists.)
- New `Scenario`: id, title, icon, settingBlurb, difficulty, raw JSON blob of variants (parse at runtime — branching trees don't fit relational modeling cleanly; keep it a codable payload), `completedCount`, `lastPlayed`.
- New `ListenEpisode`: id, title, level, durationSec, audioRefs, transcript payload (codable), questions payload, `completedCount`, `bestScore`.
- New `Passage`: id, title, style, tier, body, glossMap payload, questions payload, `read: Bool`, `lastScore`.
- New `LessonProgress` is NOT needed — concept mastery already covers Learn; scenario/episode/passage progress lives on those models.
- `DrillAxis`: confirm `production`, `comprehension`, `shadow` cover all v2 events (they do).

## 5. Mode behaviors (app)

### 5.1 Learn (centerpiece)
- Index screen: four sub-sections with visible depth — verb list with per-verb mastery tint, vocab packs 1–40 with progress, grammar lesson list, Construction entry (v1 session, works exactly as today).
- Conjugation player: table view with per-form audio → drill run (10–15 items, same prompt/reveal/grade choreography and engine as Construction — reuse SessionEngine with a single-phase configuration).
- Vocab player: pack intro (the 25 words with audio, swipeable) → sentence drill run.
- Grammar player: explanation screen (MT voice) → canonical examples with audio → drill run.
- All drill runs: reuse the existing ladder mechanics; FSRS schedules reviews across everything.

### 5.2 Speak
- Index: 12 scenario cards (icon, title, setting blurb, times completed).
- Player: chat-like staging, one exchange at a time. NPC line plays audio (street_fast; replay-slow button). User turn: English intent shown → pause to speak aloud → reveal French (street primary, formal secondary in smaller type) → self-grade → next. Branch points render as 2–3 tappable English intents. End screen: exchanges completed, accuracy, replay-with-different-variant CTA.
- Hands-free capable: with English prompt audio present, the whole flow can run screen-off except branch choices (speak-to-choose arrives with speech recognition later).

### 5.3 Read
- Index: passage cards grouped by tier, style-labeled, read/unread.
- Reader: clean typographic layout (this screen should feel like a beautifully set book page — DSType showcase). Tap any word → gloss chip appears inline. Then comprehension questions. Score updates comprehension mastery of concepts tagged to the passage tier.

### 5.4 Listen
- Index: episodes by level, best-score shown.
- Player, staged: (1) cold listen at fast speed, (2) comprehension questions, (3) transcript reveal with per-line replay + gloss, (4) optional re-listen slow, (5) optional shadow of 2 lines (reuses street-mirror mechanics).

### 5.5 Recommended path (Home top card)
- Composes daily: next Learn unit (priority: overdue FSRS reviews exist → review session; else next conjugation/grammar/vocab unit round-robin weighted by mastery gaps) + 1 Speak scenario (least-completed, difficulty-appropriate) + 1 Listen episode (level from comprehension mastery).
- Purely a suggestion — completing anything anywhere counts. No streaks, no guilt UI. A subtle "today: 2 of 3" hairline indicator at most.

## 6. Placement assessment (promoted — build early)

Runs on first launch (skippable), re-runnable from Settings. Output: MasteryScore priors per concept × axis + starting Listen level + starting Read tier.
